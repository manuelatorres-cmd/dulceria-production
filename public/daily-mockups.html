Decisions / tradeoffs

  • C · Mix (recommended). Top progress dots = whole-day overview at a glance (8 phases, done/active/upcoming). HACCP strip pinned underneath. Big focus card on current phase + its rows expanded with swatches. Below: every other phase as a collapsed accordion row (click ▸ to peek). Done phases auto-fade. Close-day pinned at bottom. Best balance — terminal flow when you want it, full visibility when you need it.
  • A · Single-column timeline. Whole day visible end-to-end. Tick boxes per row. Done phases collapse-fade. Top stat strip replaces the heavy serif "Right now" card. Side rail dropped — HACCP becomes an inline strip near close-day. Best when you bounce between phases or skip ahead.
  • B · Focused single phase. Top progress dots show all phases at a glance. Body shows ONLY the current one, large rows w/ product swatches + mould count. Big "Next phase →" button when all in this phase done. Less scroll-distraction. Best when you work strictly phase-by-phase on the workshop floor.
  • What both drop. The 320px right rail (machines · mould pool · staff · live event feed). Replaced by inline stat strip (A) or progress dots (B). Live event feed → moved to a /production-brain/log page (or Settings).
  • What both keep. HACCP entry, Close-day, per-batch ticks, Check-all-in-phase, category filter, mould count per row.
  • Click order (A): scroll → tick rows → optionally Check-all per phase → Log temps when prompted → Close day at bottom.
  • Click order (B): see active phase → tick rows in current phase → Next phase → repeat → Close day.
  • HACCP placement. A → bottom near close. B → top strip permanent. Both keep entry one-click away.
  • Mobile. A scales straight (one column already). B keeps current phase only — scrolls less.
